## 1. 产品概述

### 1.1 产品定位

Vocabuddy 是一个支持中文、英文、西里尔蒙古文的个性化词汇学习工具。首期面向中文和西里尔蒙古文母语者，支持中文→英文、中文→西里尔蒙古文、西里尔蒙古文→中文、西里尔蒙古文→英文 4 个学习方向。用户注册后设置主要母语，系统使用该母语解释所有界面、释义、学习计划和反馈；用户再选择目标语言、学习目的、当前能力和每日学习量，系统据此安排新词、间隔复习和测验。

产品的差异化能力是“跨母语好友互选单词”：两个不同国家、不同母语的好友可以基于彼此的真实交流场景，为对方推荐更合适的词语、短语、主题和学习任务。推荐内容须由接收者确认后才进入其生词本。

### 1.2 产品目标

MVP 必须验证以下假设：

1. 用户愿意完成个性化设置，并持续执行每日学习计划。
2. “新词 + 间隔复习 + 测验”比单纯浏览单词更能促成学习完成。
3. 跨母语好友推荐能提供系统难以自动判断的语境价值。
4. 好友挑战和进度对比可以提升 7 日和 30 日留存。

### 1.3 非目标

MVP 不包含：

- 正式考试报名、成绩认证或语言能力证明。
- 公开社区、公开动态、陌生人私聊。
- 教师、班级、机构后台。
- 复杂的 AI 对话口语陪练。
- 原生 iOS/Android App。
- 付费订阅和广告系统。

## 2. 目标用户与核心场景

### 2.1 主要用户

希望学习另一种语言，并且需要长期坚持的个人用户。用户可以来自不同国家，拥有不同母语和学习目标。

典型目标包括：

- 日常交流
- 旅行
- 工作沟通
- 学校学习
- 考试准备
- 阅读、影视或兴趣
- 与特定好友进行跨语言交流

### 2.2 典型用户故事

**用户 A：中文母语者学习英文**

A 设置中文为主要母语、英文为目标语言，每天学习 15 个新词。系统用中文解释英文单词，并安排复习和小测。A 的蒙古文好友可以从英文词库中挑选适合 A 工作场景的表达，A 确认后加入生词本。

**用户 B：英文母语者学习中文**

B 设置英文为主要母语、中文为目标语言。B 接收到好友推荐的中文词语时，页面、释义、例句翻译和推荐理由翻译均使用英文；中文单词、拼音/发音和例句原文保持中文。

### 2.3 用户痛点

- 不知道应该先背哪些词。
- 词书内容不能匹配真实工作、旅行或交流场景。
- 学过的单词容易遗忘，缺少系统复习。
- 学习计划过于笼统，无法知道何时达到目标。
- 独自学习容易中断。
- 朋友了解自己的真实需求，但现有工具无法让朋友直接帮忙选词。
- 跨语言好友之间的解释语言不同，容易造成沟通障碍。

## 3. 产品原则

### 3.1 母语优先

每个用户有一个主要母语。主要母语决定所有界面、系统解释、释义、学习报告、帮助文本和翻译目标语言。用户可以随时修改主要母语，修改不影响历史学习记录和掌握度。

### 3.2 目标语言保持原貌

目标语言单词、例句原文、发音和语言特征保持原语言。翻译只用于辅助理解，不替换目标语言内容。

### 3.3 推荐而非强制

好友只能为对方创建推荐任务。接收者逐词确认后，单词才进入生词本和学习计划；好友不能直接修改对方的生词本、计划或掌握状态。

### 3.4 数据驱动但不制造虚假确定性

系统根据实际学习表现更新预计达成时间。预测必须展示区间和计算前提，不能承诺用户一定达到某个语言等级。

## 4. 语言与内容范围

### 4.1 首发语言方向

首发涉及三种语言：

- 中文
- 英文
- 西里尔蒙古文

首期只支持以下 4 个语言方向：

1. 中文 → 英文
2. 中文 → 西里尔蒙古文
3. 西里尔蒙古文 → 中文
4. 西里尔蒙古文 → 英文

其中中文和西里尔蒙古文是首发主要母语，英文首期仅作为目标语言。以下方向明确不属于首期范围：

- 英文 → 中文
- 英文 → 西里尔蒙古文

每个语言方向都必须独立维护释义、例句、发音、词形变化、难度和翻译内容。不能把一个方向的本地化内容直接当作另一个方向使用。

### 4.2 内容来源

采用混合方案：

1. 自建核心词库：高频词、常用短语、基础主题、等级词。
2. 第三方词典/语言 API：补充长尾词、释义、发音、例句或词形变化。
3. 管理后台人工审核：修正翻译、例句、难度、发音和文化语境。

外部内容必须保存来源、授权状态、抓取时间和版本。第三方服务不可用时，核心词库仍必须可学习。

### 4.3 词条最低字段

- `term`：目标语言原文
- `normalized_term`：用于去重和搜索
- `language`：目标语言
- `part_of_speech`：词性
- `definitions`：按解释语言区分的释义
- `phonetic`：音标、拼音或适用的发音标记
- `audio_url`：发音地址
- `examples`：目标语言例句及翻译
- `collocations`：常见搭配
- `usage_notes`：用法说明
- `word_forms`：词形变化
- `topic_ids`：主题
- `level`：内部难度等级
- `cefr_mapping`：可选 CEFR 映射
- `source`：内容来源
- `review_status`：审核状态

## 5. MVP 功能范围与优先级

优先级定义：

- **P0**：没有该功能不能上线 MVP。
- **P1**：影响完整体验，但可以在首发后补齐。
- **P2**：长期增强功能。

### 5.1 P0-账号与认证

#### 目标

用户可以安全注册、登录，并在不同设备恢复自己的学习进度。

#### 功能要求

- 邮箱注册。
- 邮箱登录。
- 密码重置。
- 退出登录。
- 会话过期后重新认证。
- 用户注销账号。
- 注销前确认并说明数据删除影响。

#### 验收标准

```gherkin
Given 用户输入合法邮箱和密码
When 用户提交注册
Then 系统创建账号并进入母语设置流程

Given 已注册用户输入正确凭据
When 用户登录
Then 系统进入首页并加载其学习计划和进度

Given 用户输入错误密码
When 用户提交登录
Then 系统提示凭据错误且不泄露账号是否存在
```

### 5.2 P0-首次设置与学习画像

#### 必填设置

- 主要母语。
- 目标语言。
- 学习目的。
- 当前水平：未评估、初级、中级、高级或评估结果。
- 目标水平。
- 每日新词数。
- 每日可投入时长。
- 学习提醒时间，可跳过。

#### 学习目的模板

- 日常交流
- 旅行
- 工作
- 学校学习
- 考试准备
- 阅读/影视
- 自定义目标

#### 验收标准

- 未设置主要母语时不能进入核心学习页面。
- 未设置目标语言和每日学习计划时，首页展示设置引导。
- 用户可以跳过水平测试，但系统将其标记为“未评估”。
- 用户完成设置后生成一份可编辑的学习计划。

### 5.3 P0-水平评估与等级

#### 功能要求

- 提供快速词汇识别测试。
- 题目来自目标语言核心词库。
- 用户可以选择“认识/不认识”，不要求首次输入。
- 测试结束后生成初始掌握估计。
- 内部等级至少支持：Starter、Basic、Intermediate、Upper-Intermediate、Advanced。
- 可选映射 A1-C2，但不得表述为正式考试成绩。
- 学习过程中根据测验结果动态调整等级。

#### 验收标准

- 测试中途退出时保存进度，下次可继续或重新开始。
- 评估结果必须说明“这是基于词汇表现的估计”。
- 用户可以在设置中重新评估。

### 5.4 P0-学习计划与达成预测

#### 计划字段

- 母语和目标语言。
- 学习目的。
- 词库/主题。
- 当前内部等级。
- 目标等级。
- 每日新词数。
- 每日复习上限。
- 开始日期。
- 学习提醒设置。

#### 预测规则

第一版使用可解释模型：

`预计所需天数 = 目标所需有效词量 ÷ 预计每日有效掌握量`

预计每日有效掌握量由以下数据计算：

- 每日新词量。
- 近 7 日完成率。
- 近 7 日测验正确率。
- 复习完成率。
- 单词遗忘率。

系统显示预测范围，例如“按当前节奏预计 10–14 周”，并展示假设。

#### 验收标准

- 修改每日新词数后，预测日期重新计算。
- 连续 7 天存在学习数据后，预测使用真实完成率修正。
- 没有足够数据时使用默认完成率并明确标注。
- 预测不会阻塞学习功能，即使计算失败也能继续学习。

### 5.5 P0-每日学习

#### 每日任务组成

1. 到期复习词。
2. 系统安排的新词。
3. 已接受的好友推荐词。
4. 用户主动加入的历史词。
5. 今日小测。

默认流程：

`新词学习 → 到期复习 → 历史词/好友词 → 小测 → 结果总结`

#### 用户操作

- 查看单词详情。
- 播放发音。
- 标记为认识/不认识。
- 跳过单词。
- 替换单词。
- 加入生词本。
- 标记已掌握。
- 延后学习。
- 调整当天任务量。

#### 单词详情

- 目标语言单词。
- 词性。
- 母语释义。
- 音标、拼音或语言适用的发音信息。
- 发音播放。
- 目标语言例句。
- 母语例句翻译。
- 常见搭配。
- 用法说明。
- 词形变化。
- 主题、难度和等级。
- 用户笔记。

#### 验收标准

```gherkin
Given 用户存在今日学习计划
When 用户打开首页
Then 系统展示今日新词数量、待复习数量和预计时长

Given 用户完成一个新词学习卡片
When 用户选择“加入生词本”
Then 该词进入用户生词本并生成初始学习状态

Given 今日没有网络
When 用户打开已经缓存的学习内容
Then 用户仍可以完成缓存内容，联网后同步结果
```

### 5.6 P0-间隔复习与单词状态

#### 单词状态

- `new`：未学习。
- `learning`：学习中。
- `review_due`：到期复习。
- `known`：基本掌握。
- `mastered`：熟练掌握。
- `relearning`：遗忘后重新学习。

#### 复习记录

每次复习至少记录：

- 用户和词条。
- 计划和语言方向。
- 复习时间。
- 题型。
- 用户答案。
- 是否正确。
- 回答耗时。
- 用户主观难度。
- 下一次复习时间。

复习算法首版可以采用简化的 Leitner/SM-2 风格实现，但必须把算法封装为独立服务，后续可以替换。

### 5.7 P0-测验

首发支持：

- 目标语言看释义，选择单词。
- 看单词选择释义。
- 听发音选择单词。
- 输入拼写。
- 例句填空。

测验要求：

- 每次测验至少 5 题，最多 20 题。
- 新词和复习词按比例混合。
- 用户可以主动选择历史词加入测验。
- 测验结束展示正确率、错误词和下一步建议。
- 测验结果更新单词状态和整体掌握度。

### 5.8 P0-生词本与搜索

用户可以：

- 查看全部生词。
- 按语言、主题、等级、状态筛选。
- 搜索目标语言单词。
- 查看单词学习历史。
- 手动加入或移除生词。
- 添加个人笔记。
- 将历史词加入今日测验。

移除生词不会删除历史学习记录，只改变当前计划是否继续复习。

### 5.9 P0-好友与好友关系

#### 功能

- 通过用户名、邮箱或邀请码搜索用户。
- 发送好友请求。
- 接受或拒绝好友请求。
- 删除好友。
- 屏蔽好友。
- 设置好友可见数据。

#### 默认隐私规则

- 好友关系必须双向确认。
- 默认只展示今日完成度、本周学习量、连续学习天数和挑战状态。
- 详细答题记录默认不公开。
- 用户可以关闭好友提醒和好友推荐任务。

### 5.10 P0-跨母语好友选词任务

#### 创建任务

好友可以：

- 从对方目标语言词库中选择词条。
- 输入自定义单词或短语。
- 选择主题或难度。
- 写推荐理由。
- 设置建议完成日期。
- 将多个词组成一个推荐任务。

#### 推荐理由翻译

- 赠送者使用自己的主要母语填写。
- 接收者看到接收者母语的机器翻译。
- 页面同时保留原文。
- 翻译服务不可用时展示原文，不阻塞任务。

#### 接收任务

- 接收者可以逐词接受、跳过或批量接受。
- 接受后加入生词本。
- 跳过的词不会加入正式学习计划。
- 任务可整体接受、拒绝或延后处理。
- 接收者可以把自定义词标记为“内容待补充”。

#### 验收标准

```gherkin
Given 用户 A 和用户 B 已成为好友
And 用户 B 正在学习用户 B 的目标语言
When 用户 A 从词库选择单词并写下推荐理由
Then 用户 B 收到推荐任务，并看到自己的母语翻译

Given 用户 B 尚未确认推荐任务
When 用户 B 查看自己的生词本
Then 推荐词不出现在正式生词本中

Given 用户 B 接受其中一个推荐词
When 用户 B 开始下一次学习
Then 该词进入学习计划、复习算法和测验统计
```

### 5.11 P0-好友挑战与进度对比

支持：

- 双人挑战。
- 设定开始和结束日期。
- 设定每日学习量或周期总目标。
- 查看双方完成度。
- 查看挑战剩余时间。
- 挑战结束后展示结果。

不使用单纯的总词数作为唯一评价指标，同时展示完成率、复习率和连续天数。

### 5.12 P0-提醒与通知

提醒类型：

- 每日学习提醒。
- 到期复习提醒。
- 好友请求。
- 好友推荐任务。
- 挑战开始、即将结束和结束。
- 连续学习即将中断。

用户可以设置提醒时间、频率、勿扰时间、好友通知开关和邮件/站内通知开关。提醒默认关闭强打扰，仅在用户主动设置后启用。

### 5.13 P1-内容管理后台

建议作为 MVP 后半程或内部工具建设：

- 词条增删改查。
- 语言方向管理。
- 释义、例句和发音审核。
- 内容来源和授权记录。
- 翻译版本管理。
- 用户反馈处理。
- 自定义词条补全。
- 词条发布/下线。

如果没有管理后台，首发前必须准备可导入的结构化词库和人工修订流程。

## 7. 高层数据模型

### 7.1 核心实体

- `User`：账号、认证状态、主要母语、时区。
- `Language`：语言代码、显示名、书写系统、发音能力。
- `LearningProfile`：母语、目标语言、学习目的、能力等级。
- `LearningPlan`：每日任务参数、目标等级、预测结果。
- `Word`：目标语言词条和语言无关内容。
- `WordLocalization`：按解释语言存储释义、例句翻译和用法说明。
- `UserWordState`：用户对单词的状态、掌握度和下一次复习时间。
- `StudySession`：一次学习会话。
- `ReviewAttempt`：一次复习或测验答题。
- `Friendship`：好友关系状态。
- `GiftTask`：好友推荐任务。
- `GiftTaskItem`：推荐任务中的具体词条或自定义词。
- `Challenge`：好友挑战目标和时间。
- `NotificationPreference`：通知和勿扰设置。
- `VisibilityPreference`：好友可见范围。
- `ContentFeedback`：内容错误、翻译错误和发音反馈。

### 7.2 关键约束

- 所有用户学习记录必须绑定目标语言和学习计划。
- 同一个用户在不同语言方向的单词状态相互独立。
- 用户修改主要母语不删除任何学习记录。
- 好友推荐任务必须记录创建者、接收者和接受状态。
- 赠送者不能写入接收者的学习状态。
- 自定义词条必须有来源和审核状态。

## 9. 非功能需求

### 9.1 性能

- 常规页面首屏在良好网络下 2 秒内完成可交互。
- 普通 API 的 P95 响应时间小于 500ms；涉及外部词典 API 的接口 P95 小于 2 秒。
- 今日任务、已缓存单词和已开始的测验支持离线或弱网完成，联网后同步。
- 批量导入词库时不能阻塞在线学习请求。

### 9.2 安全与隐私

- 全站 HTTPS。
- 密码使用强哈希存储，不保存明文密码。
- 认证 token 具备过期和撤销机制。
- 用户只能访问自己的学习记录。
- 好友只能访问被授权的数据。
- 注销账号提供数据删除或匿名化策略。
- 推荐理由、用户笔记和好友数据不用于公开展示，除非用户明确授权。
- 记录审计日志，至少覆盖账号、隐私、好友和内容管理操作。

### 9.3 可访问性与国际化

- 支持键盘完成核心学习流程。
- 交互元素有明确可访问名称。
- 文本对比度符合 WCAG AA。
- 支持大字号和减少动画。
- 界面文本不得写死在组件中，统一进入国际化资源。
- 日期、时间和提醒使用用户时区。
- 中文、英文、西里尔蒙古文必须正确显示，不能依赖默认字体保证效果；需要指定字体回退链。

### 9.4 可观测性

必须记录：

- 登录失败率。
- API 错误率和外部服务失败率。
- 每日任务生成失败。
- 测验提交失败。
- 复习同步冲突。
- 好友推荐任务创建、接受和拒绝。
- 关键页面加载性能。

不得在日志中记录明文密码、认证 token 或未授权的私人内容。

## 10. 错误处理与边界情况

| 场景 | 前端表现 | 后端处理 |
|---|---|---|
| 无网络 | 提示当前为离线模式 | 允许缓存内容学习，结果进入同步队列 |
| 词典 API 超时 | 显示核心词条并提示部分内容暂不可用 | 超时、重试并回退到缓存/自建内容 |
| 推荐理由翻译失败 | 展示原文并标注未翻译 | 不阻塞任务创建和接收 |
| 自定义词重复 | 提示已有相同或近似词 | 不创建重复正式词条 |
| 好友已删除 | 推荐任务不可继续操作 | 任务进入关闭状态，历史记录保留 |
| 推荐者和接收者目标语言不匹配 | 阻止提交并解释原因 | 服务端再次校验语言方向 |
| 用户修改母语 | 界面和解释语言切换 | 学习记录不变，异步刷新本地化内容 |
| 重复提交答题 | 前端禁用重复按钮 | 服务端使用幂等键，避免重复计分 |
| 时区跨日 | 按用户时区计算每日任务 | 服务端统一保存 UTC，并按用户时区聚合 |
| 恶意发送任务 | 接收者可拒绝、屏蔽、举报 | 频率限制和风控记录 |

## 12. 开发分阶段计划

### Phase 0：技术验证

目标：验证语言内容和基础架构可行。

- 确认中文、英文、西里尔蒙古文的词库来源和授权。
- 验证发音、翻译、例句 API。
- 设计数据库和用户-语言方向模型。
- 验证三种文字的字体、搜索和排序。
- 形成 4 个语言方向的最小核心词库样本。

交付物：技术方案、词库样本、内容授权结论、接口草案。

### Phase 1：个人学习闭环

- 注册登录。
- 母语和目标语言设置。
- 学习目的和水平评估。
- 学习计划与预测。
- 今日任务。
- 单词详情。
- 生词本。
- 间隔复习。
- 测验。
- 基础统计。

交付标准：单用户可以从注册完成完整的一次学习、复习和测验。

### Phase 2：好友推荐与挑战

- 好友关系。
- 隐私可见范围。
- 跨母语推荐任务。
- 推荐理由翻译。
- 自定义词条。
- 双人挑战。
- 好友进度对比。
- 好友通知。

交付标准：两个不同母语用户可以互相推荐、确认、学习并比较完成度。

### Phase 3：内容管理与优化

- 内容后台。
- 内容反馈审核。
- 复习算法优化。
- 推荐质量分析。
- 学习预测校准。
- 性能、可访问性和弱网优化。

### Phase 4：候选增强功能

- 公开或小范围排行榜。
- 学习小组。
- 私聊或语音互动。
- AI 例句解释和个性化讲解。
- 教师/班级模式。
- 订阅和高级词库。
- 原生移动端。

## 13. 测试要求

### 13.1 单元测试

- 预测日期计算。
- 学习计划参数校验。
- 间隔复习算法。
- 单词状态转换。
- 测验评分。
- 好友权限判断。
- 推荐任务接受/拒绝逻辑。
- 母语切换和内容本地化选择。
- 时区和跨日计算。

### 13.2 集成测试

- 注册到完成学习计划。
- 评估结果生成计划。
- 完成测验后更新掌握度。
- 离线学习后重新同步。
- 词典 API 失败后的降级。
- 好友创建推荐任务到接收者确认。
- 推荐词进入生词本后参与复习。
- 删除好友后的权限回收。

### 13.3 端到端测试

至少覆盖：

1. 中文母语用户学习英文。
2. 英文母语用户学习中文。
3. 西里尔蒙古文母语用户学习英文。
4. 两个不同母语好友互相推荐单词。
5. 用户修改主要母语后查看历史词条。
6. 自定义词重复、未匹配和内容缺失。
7. 断网学习并恢复同步。
8. 好友屏蔽后无法继续发送任务。

### 13.4 验收门槛

- 所有 P0 验收标准通过。
- 不存在会导致学习进度丢失的已知缺陷。
- 不存在越权查看好友或他人学习记录的问题。
- 四个首发语言方向的核心词条均可完成学习和测验。
- 关键流程在桌面和移动浏览器可用。

## 14. 交付给 AI 开发工具时的实现约束

AI 开发工具必须遵守以下顺序：

1. 先搭建数据模型、认证和语言方向基础能力。
2. 先完成单用户学习闭环，再实现社交功能。
3. 所有用户输入、好友权限和推荐任务权限在服务端校验。
4. 所有用户可见文字使用国际化资源，不得在组件中硬编码中文。
5. 将复习算法、预测算法、翻译服务和词典服务封装为独立模块。
6. 外部服务必须有超时、重试、缓存和降级策略。
7. 每个 P0 功能必须同时提交自动化测试和验收说明。
8. 不得为了演示而绕过数据库、权限或学习进度记录。
9. 每完成一个阶段，必须运行测试并报告未完成事项、已知限制和下一步。
10. 任何不确定的产品行为，优先遵循本 PRD 的产品原则和验收标准，而不是自行扩大范围。

## 16. 术语表

- **主要母语**：用户用于理解系统界面、解释和学习反馈的语言。
- **目标语言**：用户正在学习的语言。
- **语言方向**：母语到目标语言的具体组合，例如中文→英文。
- **生词本**：用户确认需要学习和复习的词条集合。
- **推荐任务**：好友为用户创建的候选学习内容集合。
- **好友推荐词**：推荐任务中的单词或短语，在接收者确认后才进入生词本。
- **掌握度**：系统根据测验和复习行为计算的单词学习状态。
- **有效掌握量**：用于学习计划预测的、结合完成率和正确率修正后的每日学习量。
